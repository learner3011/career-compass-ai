Basic Admin and Support Plan

  • Priority 1: Login, OTP, payment, or full-site outage issues should be checked first.
  • Priority 2: Resume upload, report generation, and download failures should be checked next.
  • Priority 3: UI bugs, text issues, and design improvements can be handled after service-impacting issues.
  • Support requests should ask for the user email, issue summary, steps to reproduce, and screenshots if available.
  • Maintain a simple issue log with date, user email, problem type, status, and resolution notes.

Recommended Response Targets

  • Critical account or service issue: same day when possible.
  • Regular technical issue: within 1 to 2 business days.
  • Feature requests and improvement suggestions: review weekly.
